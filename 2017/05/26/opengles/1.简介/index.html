OpenGL (Open Graphics Library)一个跨编程语言，跨平台的编程接口规则的专业的图形程序接口，主要用于三维图像，一个功能强大的底层图形库

OpenGL ES (OpenGL for Embedded Systems) 是 OpenGL 三维图形 API 的子集，针对手机、PDA和游戏主机等嵌入式设备而设计。

###OpenGL　ES的使用场景:

  1. 图片处理。比如图片色调转换、美颜，滤镜等。
  2. 摄像头预览效果处理。比如美颜相机、恶搞相机等。
  3. 视频处理。摄像头预览效果处理可以，这个自然也不在话下了。
  4. 3D游戏。比如神庙逃亡、都市赛车等。

###OpenGL ES版本及Android支持情况

OpenGL ES当前主要版本有1.0/1.1/2.0/3.0/3.1。这些版本的主要情况如下：

  1. OpenGL ES1.0是基于OpenGL 1.3的，OpenGL ES1.1是基于OpenGL 1.5的。Ａndroid 1.0和更高的版本支持这个API规范。
  2. OpenGL ES 1.x是针对固定硬件管线的。
  3. OpenGL ES2.0是基于OpenGL 2.0的，不兼容OpenGL ES 1.x。Android 2.2(API 8)和更高的版本支持这个API规范。
  4. OpenGL ES 2.x是针对可编程硬件管线的。
  5. OpenGL ES3.0的技术特性几乎完全来自OpenGL 3.x的，向下兼容OpenGL ES 2.x。Android 4.3(API 18)及更高的版本支持这个API规范。

###选择OpenGL ES2.0的优势

  1. 兼容性,3.x版本向下兼容2.0,机型4.3以下手机可以使用
  2. OpenGL ES 2.0中的“顶点着色器”取代了OpenGL ES 1.x中的“变换和光照阶段”，“片元着色器”取代了“纹理颜色和环境求和”、“雾”、“Alpha测试”等阶段。虽然在一定的程度上增加了代码复杂度，但是灵活性却大大增加，同时也能够处理OpenGL ES 1.x中难以完成的处理任务。
